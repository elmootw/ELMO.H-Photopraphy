# 📸 ELMO.H Photography 作品集網站 - 專案需求總覽
## 專案識別與品牌風格
* 品牌與標題
    * 網站標題： ELMO.H Photopraphy
* 整體視覺風格：經典、優雅 (Classic & Elegant)
    * 字體： Serif Fonts (襯線字體)
    * 主色 (背景)： 象牙白、暖白或奶油色 (提供溫暖、柔和的背景)
    * 輔助色 (文字/標題)： 酒紅色
    * 強調色 (點綴)： 玫瑰金 (用於網站元素，如 Logo、裝飾線條)

## 技術棧與部署

核心技術： JavaScript
前端框架： React 函式庫
樣式框架： Tailwind CSS
響應式設計： 必須具備 RWD (Responsive Web Design)
檔案結構： 採用現代前端結構 (src/components, src/pages 等)
部署環境： Github Pages
部署流程： 使用 Github Actions 進行自動化部署

## 作品集內容與分類結構

### 頂級分類

個人 (/personal)
情侶 (/couple)
登記 / 婚宴 (/wedding)
活動 (/event)
生活 (/lifestyle)
街拍 (/street)
風景 (/landscape)
動態 (/motion)

### 內容組織要求

動態內容： 要保留會持續新增相簿或照片、刪除相簿照片、每個相簿照片張數不同的寬容度，可接受透過重新部署實現。
相簿名稱： 資料夾名稱。
相簿形狀： 相簿封面縮圖必須是正方形。
照片形狀： 維持原始照片形狀。
相簿封面： 每個相簿資料夾內需要有 `cover.jpg`。
類別封面： 放在 `public/images/covers/` 下，命名為 `{分類名}.jpg`（如 `personal.jpg`、`wedding.jpg` 等）。

### 特殊分類要求

生活 (/lifestyle)： 需建立 Hashtag 標籤，並可在該頁面進行篩選。
街拍 (/street)： 僅放置照片，不需相簿/子分層結構。
動態 (/motion)： 嵌入 YouTube 影片。

## 圖片處理與互動功能

圖片載入： 圖片必須採用非同步載入 (Lazy Loading) 模式。
圖片展示： 照片點擊後以 Pop-out (Modal) 方式放大顯示。
安全保護： 不能複製下載（需禁用右鍵菜單等下載途徑）。
